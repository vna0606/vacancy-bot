# 03-tribute-webhook — приём вебхуков Tribute о донатах

## Зона ответственности

Принимает уведомления об оплате от Tribute (фича «Донаты», настраивается вручную через
@Tribute бота — донат-страница создаётся там, ссылка на неё лежит в `01-bot` как
`TRIBUTE_DONATE_URL`). Этот этап НЕ создаёт донат-страницы и не выбирает сумму/периодичность —
это делает донатер сам в интерфейсе Tribute.

- `POST /api/webhook` — принимает события `new_donation`, `recurrent_donation`, `cancelled_donation`
- Проверяет подпись `trbt-signature` (HMAC-SHA256 от raw body на `TRIBUTE_API_KEY`) — без
  валидной подписи запрос отбрасывается с 401
- Пишет каждое событие в таблицу `donations` (Turso) — дедуп повторной доставки через
  UNIQUE-ограничение
- Если донат не анонимный — шлёт донору личное сообщение-благодарность через Telegram Bot API
  (прямой HTTP-вызов, без библиотек)

Почему отдельный этап, а не часть `01-bot/`: другой стек (Node.js, не Python) и другая
инфраструктура (Vercel serverless, не процесс на AWS-сервере) — Tribute требует публичный
HTTPS-адрес для вебхука, на сервере `01-bot` такого нет, а Vercel даёт его бесплатно из коробки.

## Входной контракт

Нет входного контракта от других этапов проекта. Реагирует на вебхуки Tribute (внешний сервис).

## Выходной контракт (Turso)

### Таблица `donations` (пишет и читает только этот этап)

Полная схема — в корневом `CLAUDE.md`. Создаётся функцией самостоятельно
(`CREATE TABLE IF NOT EXISTS`) при первом запросе — отдельного шага миграции не требуется.

## Структура файлов

```
03-tribute-webhook/
  api/
    webhook.js      — POST /api/webhook: проверка подписи, парсинг payload, запись в Turso,
                       благодарность донору
  package.json      — без зависимостей, только встроенные crypto/fetch (Node 24.x)
  .env.example       — список переменных (реальные значения — в Vercel env, не в файле)
  vercel.json (если появится) — конфигурация Vercel-проекта
```

## Принципы реализации

- Без npm-зависимостей: HMAC через встроенный `crypto`, HTTP-вызовы через глобальный `fetch`
  (доступен в Node 18+)
- `module.exports.config = { api: { bodyParser: false } }` — обязательно: подпись считается от
  RAW-байт тела запроса, автоматический JSON-парсинг Vercel сломал бы проверку подписи
- Запись в Turso — тот же HTTP pipeline API, что использует `01-bot/db.py`, но независимая
  реализация на JS (нет общего кода между этапами — по правилам проекта)
- Telegram-сообщение донору — прямой `fetch` на `api.telegram.org`, без aiogram/SDK (процесс
  на Vercel — короткоживущий, поднимать полноценного бота избыточно)
- Деплой: `vercel deploy --prod` из этой папки (используется `VERCEL_TOKEN` из корневого
  `/home/ubuntu/claude-bot/workspace/.env`, аккаунт `vna0606`, проект `03-tribute-webhook`
  в team `nikitas-projects-c3892610`)
- Production URL: `https://03-tribute-webhook.vercel.app/api/webhook` — именно этот адрес
  нужно указать в настройках Tribute (Панель автора → API-ключи → Webhook URL)

## СТРОГИЕ ЗАПРЕТЫ

- НЕ читай файлы из `01-bot/` или `02-notifier/` напрямую — только через таблицу `donations`,
  если когда-нибудь понадобится статистика донатов внутри бота
- НЕ меняй схему таблицы `donations` без обновления корневого `CLAUDE.md`
- НЕ включай автоматический body-parser Vercel — сломает проверку подписи
